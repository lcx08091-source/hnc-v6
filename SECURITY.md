# 安全漏洞报告

如果你发现 HNC 可能存在安全漏洞,**请不要**在公开 issue 里报告。

## 报告渠道

邮件:**lcx08091@gmail.com**

主题:`[SECURITY] HNC - <简述>`

内容建议包含:
- 影响版本
- 复现步骤(代码 / 命令)
- 预期影响
- 建议修复(如果有)

## 响应时间

- 24 小时内确认收到
- 7 天内给出初步影响评估
- 严重漏洞:出 hotfix,然后在 release note 公开 + 致谢

## 关注的范围

以下是**在报告范围内**的(欢迎报告):

- 远程 WebUI 可被未授权访问 / 绕过 auth
- token 泄漏 / 伪造
- 远程代码执行
- 提权(非 root 用户通过 HNC 拿到 root)
- 信息泄漏(读取其他客户端数据)

以下**不在报告范围**(已知):

- 受信任用户通过合法 token 做"他本来就有权做的事"
- root 被攻破后 HNC 被改 —— 这不是 HNC 的威胁模型
- Android framework / Magisk / 内核本身的漏洞(请报告给对应项目)
